# Руководство по конфигурации для клиента — Цветков Мебель

В данном руководстве описаны все ключевые настройки системы, управление переменными окружения и интеграция с внешними сервисами.

---

## ⚙️ Файл переменных окружения (`.env`)

Все системные настройки задаются в файле `.env` в корневой директории проекта:

```ini
# Строка подключения к базе данных (SQLite по умолчанию)
DATABASE_URL="file:./dev.db?connection_limit=1&socket_timeout=10&busy_timeout=10000"

# Данные авторизации Администратора (CEO)
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="admin"

# Данные авторизации Менеджера CRM
MANAGER_USERNAME="manager"
MANAGER_PASSWORD="manager_password"

# Секретный ключ для подписи JWT сессий (смените в продакшене!)
JWT_SECRET="f657519c23daee828731ad2b7245b0aef1c05d762f9bb6c4dbbdff4497cd9215"

# Интеграция с Telegram-ботом для мгновенного оповещения о лидах
TELEGRAM_BOT_TOKEN="your_bot_token_here"
TELEGRAM_CHAT_ID="your_chat_id_here"
```

## 📢 Настройка оповещений в Telegram

Чтобы настроить оповещения о новых заявках в ваш рабочий Telegram-чат:

1. **Создайте Telegram-бота**:
   * Напишите в Telegram боту `@BotFather`.
   * Отправьте команду `/newbot`, задайте имя и логин для бота.
   * `@BotFather` выдаст вам токен вида: `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`. Скопируйте его в `TELEGRAM_BOT_TOKEN`.
2. **Получите ID чата (TELEGRAM_CHAT_ID)**:
   * **Для личного чата**: Напишите боту `@userinfobot` или вашему созданному боту, затем перейдите по адресу `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` и найдите поле `chat.id`.
   * **Для группы/канала**: Добавьте вашего бота в группу в качестве администратора. Напишите любое сообщение в группу, перейдите по ссылке `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` и скопируйте `chat.id` (он начинается с минуса, например: `-100123456789`).
3. **Обновите файл `.env`**:
   * Вставьте токен и ID чата в файл `.env`.
   * Перезапустите сервер Next.js. Оповещения начнут приходить мгновенно при каждой новой отправленной форме!

## 🔒 Настройка безопасности перед деплоем

При переносе проекта на публичный хостинг обязательно выполните следующие действия:

1. Измените имя администратора и менеджера в `.env` на уникальные значения.
2. Сгенерируйте новый длинный секретный ключ для JWT:
   ```bash
   # Вы можете сгенерировать случайный шестнадцатеричный ключ в терминале Node:
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```
   Вставьте полученное значение в поле `JWT_SECRET`.

## 🐘 Настройка PostgreSQL в облаке

Для развертывания базы данных на таких платформах как Yandex Cloud, Supabase, Neon или VPS:

1. Создайте базу данных PostgreSQL.
2. Получите Connection String (строку подключения), которая выглядит следующим образом:
   `postgresql://имя_пользователя:пароль@хост:5432/имя_базы?sslmode=require`
3. Установите ее в значение `DATABASE_URL` в панели переменных окружения вашего хостинг-провайдера (например, Vercel, Amvera или VPS).
4. Прогоните миграцию базы данных с помощью команды:
   `npx prisma db push`
   (команда автоматически воссоздаст все таблицы: `Product`, `Promo`, `Review` и `Lead` со всеми нужными полями и типами данных).
